## 1. 引言

### 1.1 编写目的

本需求规格说明书旨在详细描述 KOLBox 网红营销管理系统的功能需求、业务流程和数据关系，作为系统设计、开发、测试和验收的依据。

### 1.2 项目背景

**项目名称**：KOLBox 网红营销管理系统

**业务背景**：
KOLBox 是一个面向网红营销（KOL Marketing）领域的综合管理平台，服务于品牌方和营销团队。系统核心目标是整合 TikTok、Instagram、YouTube 三大主流社媒平台的网红资源，通过产品关联和内容类型两条业务链条，实现网红资源的智能筛选、分配和开发管理。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| KOL | Key Opinion Leader，关键意见领袖，即网红、达人 |
| 标签 | 用于分类网红的关键词，来源于话题标签、搜索关键词、种子视频、种子账号 |
| 达人类型 | 网红的内容创作类型分类，如美妆、游戏、时尚等 |
| 建联 | 与网红建立联系的过程，主要通过邮件营销实现 |
| 商家 | 品牌方客户，通过商家端发起推广需求 |
| 商务人员 | 负责对接商家、处理推广需求的内部人员 |

## 2. 系统概述

### 2.1 系统目标

建立高效、智能的网红营销管理平台，实现：

1. **资源整合**：整合三大平台网红资源，建立统一资源库
2. **智能筛选**：通过产品关联和内容类型精准匹配网红
3. **分配管理**：将网红资源合理分配给顾问，记录分配历史
4. **营销自动化**：批量邮件发送、效果追踪
5. **推广流程管理**：推广需求全流程管理

### 2.2 支持平台

- Instagram
- YouTube
- TikTok

### 2.3 用户角色

| 角色 | 职责 |
|------|------|
| 超级管理员 | 系统配置、用户管理 |
| 运营主管 | 资源分配、团队管理 |
| 网红顾问 | 网红筛选、开发、跟进 |
| 商务人员 | 商家对接、推广需求处理 |
| 商家用户 | 发起推广需求 |

## 3. 业务流程

### 3.1 网红开发主流程

```
资源采集 → 标签分类 → 筛选分配 → 建联开发 → 合作跟进 → 效果追踪
```

**流程说明**：
1. 资源采集：录入网红基础信息（TK/IG/YT数据）
2. 标签分类：关联标签/品牌/产品，建立关联关系
3. 筛选分配：按条件筛选并分配给顾问，记录分配日志
4. 建联开发：批量邮件发送，建立联系
5. 合作跟进：回复状态跟进，推进合作
6. 效果追踪：推广数据统计

### 3.2 网红分配流程

```
开始 → 设置筛选条件 → 查询网红列表 → 选择目标顾问 → 执行分配 → 记录日志 → 结束
```

**筛选条件**：
- 选择平台（TK/IG/YT）
- 选择品牌+产品
- 或选择达人类型
- 或直接选择标签

**分配记录**：
- 记录分配类型
- 记录分配内容
- 更新分配状态

### 3.3 分配状态

| 状态名 | 说明 |
|--------|------|
| 分配中 | 分配任务正在执行 |
| 分配完成 | 分配任务成功完成 |
| 分配异常 | 分配过程中发生错误 |

### 3.4 分配类型

| 类型名 | 分配依据 |
|--------|----------|
| 标签分配 | 按标签名称分配 |
| 产品分配 | 按产品关联的标签分配 |
| 达人类型分配 | 按达人类型分配 |

## 4. 核心业务链条

### 4.1 产品-标签-网红链条

通过品牌、产品与标签的关联，实现从产品到网红的精准匹配。

**链条流程**：
```
品牌 → 产品 → 标签 → 网红 → 分配给顾问
```

**业务场景**：
- 查找某产品的网红：选择品牌 → 选择产品 → 查找关联标签 → 筛选网红
- 为品牌匹配网红：选择品牌 → 查找品牌关联标签 → 筛选网红

### 4.2 达人类型-标签-网红链条

通过达人类型（内容分类）与标签的关联，实现按内容类型筛选网红。

**链条流程**：
```
达人类型 → 标签 → 网红 → 分配给顾问
```

**业务场景**：
- 按达人类型筛选网红：选择达人类型（如"美妆"）→ 查找关联标签 → 筛选网红
- 产品达人类型筛选：选择产品 → 选择产品的达人类型 → 查找标签 → 筛选网红

### 4.3 两条链条的关系

产品标签链和达人类型链可以独立使用，也可以组合使用，实现更精准的筛选。

**筛选条件互斥规则**：

| 条件组 | 包含条件 | 互斥关系 |
|--------|----------|----------|
| 标签组 | 标签 | 与产品组、达人类型组互斥 |
| 产品组 | 品牌 + 产品 | 与标签组互斥；与达人类型组不互斥 |
| 达人类型组 | 达人类型 | 与标签组互斥；与产品组不互斥 |

**组合筛选示例**：
- 只选标签：按标签筛选
- 只选达人类型：按内容类型筛选
- 产品 + 达人类型：同时满足产品和内容类型要求
- 标签与产品/达人类型：不能同时选择

- 视频样本统计
- 备注
- 分配状态

### 5.3 F02 网红筛选分配

#### 5.3.1 功能描述

基于产品关联链或达人类型链筛选网红，并将结果分配给顾问。

#### 5.3.2 功能列表

| 功能编号 | 功能名称 | 功能描述 |
|----------|----------|----------|
| F02-001 | 平台选择 | 选择目标平台（TikTok/Instagram/YouTube） |
| F02-002 | 品牌+产品筛选 | 通过品牌→产品→标签链条筛选网红 |
| F02-003 | 产品类目筛选 | 通过产品类目筛选网红 |
| F02-004 | 达人类型筛选 | 通过达人类型→标签链条筛选网红 |
| F02-005 | 标签直接筛选 | 直接选择标签筛选网红 |
| F02-006 | 更多筛选条件 | 粉丝数、视频数、国家等附加条件 |
| F02-007 | 批量分配 | 将筛选结果分配给顾问 |
| F02-008 | 分配记录查询 | 查看网红分配历史 |

#### 5.3.3 筛选条件

**主要筛选条件**：
- 平台类型
- 品牌 + 产品
- 产品类目
- 开发状态（未开发/已开发）
- 国家
- 私有网红（是/否）

**更多筛选条件**：
- 达人类型
- 粉丝数范围
- 有效视频数
- 视频播放中位数
- 平均播放量
- 标签
- 账号
- 邮箱
- 分配日期

#### 5.3.4 筛选条件互斥规则

筛选条件分为三个互斥组：

| 组别 | 包含条件 | 互斥关系 |
|------|----------|----------|
| 标签组 | 标签 | 与产品组、达人类型组互斥 |
| 产品组 | 品牌 + 产品 | 与标签组互斥；与达人类型组不互斥 |
| 达人类型组 | 达人类型 | 与标签组互斥；与产品组不互斥 |

#### 5.3.5 分配操作

**分配流程**：
1. 选择目标顾问（可多选）
2. 创建分配日志，状态设为"分配中"
3. 更新网红的顾问ID和分配状态
4. 创建顾问关联记录
5. 创建分配明细记录
6. 更新分配日志状态为"分配完成"

**分配日志记录内容**：
- 平台类型
- 分配类型（标签/产品/达人类型）
- 分配内容描述
- 分配顾问列表
- 分配时间范围
- 分配状态

#### 5.3.6 业务规则

| 规则编号 | 规则描述 |
|----------|----------|
| BR-F02-001 | 选择品牌+产品时，系统自动查找产品关联的标签进行筛选 |
| BR-F02-002 | 选择达人类型时，系统自动查找达人类型关联的标签进行筛选 |
| BR-F02-003 | 分配时自动跳过已分配的网红 |
